What makes this Design Fast?

First of all, you can write faster particle systems than this on the CPU.

This is not the fastest way to write particle systems. This design does however exist in a
comfortable spot between 'simple' and 'fast enough'. I would argue that this type
of Particle System design is a good balance of easy to use and fast to update + draw.

But what makes it fast? That is primarily due to using small Structs in Arrays, and iterating
them sequentially. You will also notice that this design generates very little garbage. In fact,
the only real garbage it creates is the string that is written to the title of the window for
timing purposes. All other data either lives within a stack frame, or is stored on the heap
in a public static class, which means it's never garbage collected.
